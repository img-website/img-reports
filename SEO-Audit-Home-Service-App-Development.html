3. Schema / Structured Data — needs attention

Detected: Product (+AggregateRating 4.9/140), @graph [Organization, WebSite, WebPage], Service, FAQPage (DB), BreadcrumbList (DB). Most schema types of any page — but two are risky.

CRITICAL 3.1 — AggregateRating not visible on page (policy risk)

Product schema declares:

"aggregateRating": { "ratingValue": "4.9", "ratingCount": "140" }

Verified: "4.9" and "140" appear nowhere in the live page content. Google's structured-data policy requires ratings to be visible to users and reflect genuine reviews. Invisible/unverifiable ratings can trigger a "Structured data issue" manual action or simply be ignored — and risk the whole rich-result eligibility for the domain. It also contradicts the "4.5/5" shown on every other page (inconsistent rating data across the site).

Solution (do not apply yet): either (a) display the genuine rating + review count visibly on this page and back it with real, on-page reviews, or (b) remove the AggregateRating. Make the rating value consistent with the rest of the site. Never hard-code a rating that isn't shown and substantiated.
HIGH 3.2 — Product schema used for a service

"Home Service App Development" is a service, not a Product. A Service node already exists on the page. Using Product purely as a vehicle for the rating is the wrong type and compounds 3.1.

Solution: drop the Product node; if you keep a rating, attach a truthful aggregateRating to the Service or Organization node, backed by visible reviews.
MEDIUM 3.3 — Organization sameAs empty + missing dates
@graph Organization has sameAs: [] (Service node has the socials — populate Organization too). WebPage node lacks datePublished/dateModified — add them.
