2 The only 3 genuine issues (and exactly where each lives)

🔴 CRITICAL — "Meta Title:" / "Meta Description:" label leaked into blog #8

Where: /blog/mobile-app-development-companies-in-india — the only blog with 2,414 impressions, stuck at position 71.
What: The page title, the JSON-LD headline, and the meta description literally read "Meta Title: Top Mobile App…" and "Meta Description: …" — the authoring labels were pasted into the fields. Title is 86 chars, meta-desc 268 chars (both over limit).
How it arose: This is a DATA / CMS mistake, not code. The blade is correct — blogDetail.blade.php:113 just echoes {{$blog->meta_title}}. The database row for this blog has "Meta Title: …" stored verbatim in its meta_title and meta_description fields.
Fix (in your blog admin — no code):
Title → Top Mobile App Development Companies in India 2026 | Compare Firms (remove "Meta Title:", keep ≤ 60 chars)
Meta description → remove the "Meta Description:" prefix and trim to ≤ 160 chars.
Why it matters: This is your highest-impression blog. A title that opens with "Meta Title:" tanks click-through and signals low quality to Google — fixing it is the single highest-ROI change in this whole list.

🟡 MEDIUM — HowTo schema misused on 7 blogs (deprecated, dead weight)

Where: blogs #1–#7 (the WordPress posts). Each emits a HowTo block with 20–36 "steps".
What: The "steps" are actually article section headings ("What Is a Klarna App?", "Why Businesses Invest…") — not real instructions. Google retired HowTo rich results in Sept 2023, so these earn zero benefit and just bloat the markup. (Not a penalty, not a ranking blocker — just useless.)
How it arose: CODE — the blog schema partial auto-generates HowTo from H2/H3 headings.
Fix (code, optional cleanup): Remove the HowTo block from the blog schema template, or only emit it when the content genuinely has numbered steps. Keep BlogPosting + FAQPage.

🟡 MEDIUM — Two LocalBusiness blocks on every blog page

Where: all 10 blogs — each carries a Jaipur and a Dubai LocalBusiness entity.
What: LocalBusiness belongs on the homepage / contact / about — not on every blog article. It's wasteful and slightly confusing to crawlers (no penalty; no fake ratings present).
How it arose: CODE — global schema partial included on all pages.
Fix (code, optional): Keep LocalBusiness on home/contact/about; drop it from blog detail templates. Keep Organization.

3 The cache — what it actually is, and the only real gotcha

Read straight from config/responsecache.php.

The site uses Spatie ResponseCache — it stores the full rendered HTML of each page and serves it from a file cache for speed.

  • Why "poisoning" is impossible: the cache key is a hash of the full request URL (DefaultHasher, line 65). So /blog/A and /blog/B get separate cache entries — one blog's HTML can never be served on another blog's URL. My 3× live fetches confirmed this in practice.
  • The ONE real gotcha: the cache lifetime is 7 days (line 22). So when you edit a blog (DB or blade), the old HTML keeps serving for up to 7 days until the cache is cleared. That's the "stale content" effect — it's not wrong content, just outdated content.

How to fix / handle the cache

# after editing any blog, clear the response cache: php artisan responsecache:clear # (already wired into the clearLaravelCache endpoint — it now also runs # view:clear + ResponseCache::clear(), which were previously commented out)

After clearing, the very next request re-renders the page fresh and re-caches it. This is the entire "cache issue" — a clear-after-edit ops step, nothing more.
