🧩 Alag se mile (is page ka SEO defect NAHI — site-wide/footer/manual)

Ye cheezein on-demand page ke content/SEO ka part nahi hain — shared global components ya GSC ka manual action hain. Transparency ke liye likh raha hoon.

A. Footer me Germany office pe galat image Footer · site-wide
Shared footer (components/footer/address/address.blade.php) me "Germany (Berlin)" office ke saath galti se Canada ka flag/HQ image (canadaFlag.webp / hq-canada.webp) laga hai. Maine CDN check kiya — germanyFlag.webp / hq-germany.webp exist hi nahi karte (403), isliye safely fix nahi kar sakta (broken image ban jaata). Aap correct Germany images CDN pe daal dein to 30-sec me theek kar dunga. (Ya batayein Berlin office actual hai ya entry hatani hai.)
B. Shared-component images me width/height attribute missing site-wide · low
24 images (rating stars SVG, mail icon, offices section, lead-popup) — sab shared/global components me hain, page ke content me nahi. Inpe CSS me fixed-size (size-3, h-40) ya aspect-[..] set hai → CLS already controlled. Page ke apne content images me width/height 100% set hain. Chahein to global components me bhi explicit dimensions daal dun (poori site ko fayda).
C. GSC — Request Indexing Aapka manual action
Purana .php URL abhi index me hai. Clean URL ke saare signals sahi hain (301, canonical, sitemap). Bas GSC → URL Inspection → Request Indexing dabana hai. Ye GSC UI action hai — main automate nahi kar sakta (Windsor read-only).
