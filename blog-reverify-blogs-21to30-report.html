2 The genuine issues (each independently verified)

🟡 MEDIUM — author.url = the author's PHOTO, not a profile page (ALL 10)

Where: every blog #21–30 (deterministically verified on all 10).
What:
"author": { "@type": "Person", "name": "…", "url": "https://d1y41eupgbwbb2.cloudfront.net/images/blogauthor/blogauthor-1747657504.webp" } // url must be the author's bio PAGE, not a .webp photo
Effect: Google can't resolve the author entity → weaker author / E-E-A-T signal.
How it arose: CODE — the blog schema partial fills author.url with the author image field.
Fix: point author.url to a real /author/{slug} page; keep the photo in author.image. (code — one fix solves all blogs)

🟡 MEDIUM — HowTo schema misused on 7 blogs, with empty "garbage" steps on 3

Where: #22 snapchat, #23 temu, #24 educational, #26 tiktok, #27 magicbricks, #28 makemytrip, #29 rapido.
What: "Steps" are article section headings ("What Is a Snapchat App?", "Property Listings"), not real instructions. Google retired HowTo rich results in Sept 2023, so they earn nothing. Worse — verified empty/whitespace-only steps: makemytrip ×3, temu ×1, educational ×1 (step name is just a non-breaking space → garbage data in schema).
How it arose: CODE — schema partial auto-builds HowTo from H2/H3 headings (and emits blank ones where a heading was empty).
Fix: remove the HowTo block from blog schema (keep BlogPosting + FAQPage). (code)

🟡 MEDIUM — HTML entities leaking raw into JSON-LD (verified on 3 blogs)

Where: #30 dream11 headline + description; #23 temu & #28 makemytrip descriptions.
What: The headline/description contain HTML entities instead of real characters:
#30 headline: "Fantasy Sports App Development Like Dream11: Cost &amp; Features 2026" // Google reads the literal text "&amp;" — not "&". Shows wrong in rich preview.
How it arose: CODE — values are HTML-encoded before being injected into JSON-LD (they should be decoded; JSON-LD is not HTML).
Fix: decode HTML entities (html_entity_decode()) before outputting any value into the JSON-LD script. (code)

🟡 MEDIUM — meta descriptions over 160 chars on 7 blogs

Where & verified lengths: #23 (172), #24 (209), #26 (223), #27 (180), #28 (247), #29 (203), #30 (197).
Effect: Google truncates them mid-sentence in results, hurting click-through. #28 (247) and #26 (223) are the worst.
Fix: trim each to ≤ 155–160 chars, benefit + CTA up front. (content)
