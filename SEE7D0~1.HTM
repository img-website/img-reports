3. Schema / Structured Data — needs attention

Detected: Product (+AggregateRating 4.9/140), @graph [Organization, WebSite, WebPage], Service, FAQPage (DB), BreadcrumbList (DB).

CRITICAL 3.1 — AggregateRating not visible on page (policy risk) 
"aggregateRating": { "ratingValue": "4.9", "ratingCount": "140" }

Verified: "4.9" / "140" appear nowhere in the live page. Identical hard-coded values to the Home Service page → looks like a templated, unsubstantiated rating. Google requires visible, genuine ratings; this risks a structured-data manual action and contradicts the site-wide "4.5/5".

Solution: show a genuine, on-page rating backed by real reviews, or remove the AggregateRating. Make rating values consistent across the site. Fix this on the shared on-demand template so all sibling pages are corrected at once.
HIGH 3.2 — Product schema used for a service
Drop the Product node (a Service node already exists). If keeping a rating, attach a truthful aggregateRating to Service/Organization with visible reviews. Also the Product name ("Dating App Development Agency | Dating App Development Firm") is keyword-stuffed with a pipe — use a clean name.
MEDIUM 3.3 — Organization sameAs empty + missing dates
Populate @graph Organization sameAs; add datePublished/dateModified to WebPage.
