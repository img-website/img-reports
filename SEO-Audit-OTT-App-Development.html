1. Technical SEO

Verified correct

  • INDEX, FOLLOW (live), self-referencing canonical, single H1, viewport, WebP/CDN, server-rendered.
  • 60 images with dimensions; lower inlne-flex count (12) than sibling pages.
MEDIUM 1.1 — Two eager + fetchpriority="high" hero images (LCP competition)

Both ottHeroImage.webp and the large ottBgImage.webp (1526×516) are loading="eager" fetchpriority="high". Two high-priority fetches compete for bandwidth and can delay the true LCP element.

Solution: keep fetchpriority="high" on only the actual LCP image (the hero), and set the background image to fetchpriority="low" / default (the homepage already documents this pattern correctly).
MEDIUM 1.2 — robots.txt has no Sitemap directive; inlne-flex typo (12×)
Solution: add Sitemap: line; fix inlne-flex → inline-flex (shared component).
LOW 1.3 — llms.txt served but stale (.php URLs)
Deploy the clean-URL repo version.

3. Schema / Structured Data — weakest on the site

Detected: Product (+AggregateRating 4.9/140), FAQPage (DB), BreadcrumbList (DB). Missing the WebPage, WebSite, Organization (@graph) and Service nodes that every other service page includes.

CRITICAL 3.1 — Rating mismatch + invisible/templated AggregateRating 
Schema says:  "ratingValue": "4.9", "ratingCount": "140"
Page shows:   "4.5/5 Average Rating"

The structured-data rating (4.9/140) does not match the rating visible on the page (4.5/5), and 4.9/140 is the same hard-coded value found on the Home Service and Dating pages — i.e. templated and unsubstantiated. Google requires structured-data ratings to match visible, genuine ratings; a mismatch is a clear policy violation and rich-result risk.

Solution: make the schema rating equal the genuine visible rating, backed by real reviews — or remove the AggregateRating. Fix on the shared on-demand template so all sibling pages are corrected together.
HIGH 3.2 — Missing core schema (WebPage / Service / Organization)
Unlike the other service pages, this page has no WebPage/Service/Organization @graph. Add them (mirroring the React Native / Flutter pages) so the page declares its entity, service type, breadcrumbs context, and dates. This is a notable consistency gap.
HIGH 3.3 — Product schema for a service
Drop the Product type; if keeping a rating, attach a truthful aggregateRating to a proper Service node.
