2 The genuine issues (each independently verified)

🔴 CRITICAL — stray TAB breaks the BlogPosting schema on blog #16

Where: /blog/building-a-multi-vendor-marketplace-app-like-amazon (545 impressions, position 51).
What: The JSON-LD headline ends with a literal TAB character:
"headline": "How to Create Online Marketplace like Amazon? Complete Guide\t", // a raw tab (char code 9) is an illegal control char inside a JSON string (RFC 8259)
Verified: I extracted the headline directly — hasTAB=True, lastCharCode=9. Confirmed by the independent schema agent (JSON parse fails: "invalid control character").
Effect: Google's parser rejects the entire BlogPosting block — so this post loses its article schema, author, and dates in structured data. (Title still shows fine to users; the loss is the structured-data signal.)
How it arose: a trailing tab was pasted into the blog's title field in the CMS; the schema template injects it verbatim.
Fix: remove the trailing tab from the title (data — edit in blog admin), and as a safety net add trim() when the schema outputs the headline (code).

🟡 MEDIUM — author.url points to the author's PHOTO, not a profile page (all DB blogs)

Where: every database blog — verified on #13, #14, #15, #17, #20 (and applies to #18, #19 too).
What: The author's url in BlogPosting is a CloudFront image file:
"author": { "@type": "Person", "name": "…", "url": "https://d1y41eupgbwbb2.cloudfront.net/images/blogauthor/blogauthor-1747657526.webp" } // url should be the author's profile/bio PAGE, not a .webp image
Effect: Google can't resolve the author entity → weaker author/E-E-A-T signal (which matters for "cost/guide" YMYL-adjacent topics).
How it arose: CODE — the blog schema partial fills author.url with the author image field instead of a bio-page URL.
Fix: point author.url to a real author page (e.g. /author/{slug}) and keep the photo under author.image. (code)

🟡 MEDIUM — HowTo schema misused on 4 blogs (deprecated, dead weight)

Where: #14 digital-gold, #16 marketplace-amazon, #18 dating, #19 taxi-booking.
What: The "steps" are article section headings ("Market Overview…", "Aggregator Model", "What is Dating app?") — not real instructions. Google retired HowTo rich results in Sept 2023, so these earn nothing and just bloat the markup. Not a penalty.
How it arose: CODE — schema partial auto-builds HowTo from H2/H3 headings.
Fix: remove the HowTo block (or only emit it for genuine step content). Keep BlogPosting + FAQPage. (code)

🟡 MEDIUM — meta descriptions over 160 chars on 5 blogs (truncate in SERP)

Where & verified lengths: #13 (170), #14 (175), #15 (180), #16 (167), #20 (171).
Effect: Google cuts them off mid-sentence in results, hurting click-through.
Fix: trim each to ≤ 155–160 chars with the key benefit + a call to action up front. (content)
