6. Cannibalization — what's real vs what's noise

Real & realized (fix now): the .php duplicate. /ai-development-company AND /ai-development-company.php both rank for the same queries (e.g. "img global infotech": pos 4.0 [.php, 161 impr] vs 4.7 [clean, 121 impr]). The legacy .php is actually ranking better and stealing impressions. Fix: 301-redirect every .php URL → its clean URL.

Latent on-page cannibalization (fix before they start ranking): the 6 service pages share templated intros + the same case-study block, so they read as generic "AI development". They don't yet collide in the SERP only because none ranks well — but they will compete as they gain authority. Rescope now:

Competing pagesHow to differentiate
ai-development-company ↔ generative-aiKeep head term "AI development company India" on the hub only; rescope GenAI to generative/GenAI intent (image/text/audio, RAG, content gen).
generative-ai ↔ LLMGenAI = outcomes/use-cases; LLM = model engineering (custom training, fine-tuning, eval, MLOps). Cross-link as siblings.
ai-agent ↔ ai-chatbotChatbot = support/FAQ bots, channel deploy; Agent = autonomous/agentic, multi-step tool use. Add a "chatbot vs AI agent" block on each.
LLM ↔ machine-learningML = broad parent (classical ML, predictive); route deep-learning/LLM intent to the LLM page.

Note: the many "img global infotech" / site: queries that hit multiple pages are normal brand/navigational behaviour — not a cannibalization problem; ignore them.

9. Issues by dimension (full detail)

Meta

  • Title missing India/intent keyword (9 pages). Fix format: [Primary Keyword] in India | IMG Global Infotech.
  • Over length: Machine Learning title 63; AI Chatbot meta 168; Generative AI & AI in Education metas at exactly 160 (no room for CTA).
  • title vs og:title vs twitter:title mismatch on most pages — LLM's twitter:title is "Top Machine Learning Development Company" (wrong service). Align all four (incl. schema name) per page.
  • Meta typos: Generative AI ends "…today!."; AI in Healthcare trailing space + redundant meta name="title".
  • No CTA/geo in meta: AI in Banking, Education, Retail.

Schema

  • Duplicate Organization (×3) on 9/11; name mismatch IMGGlobalInfotech vs IMG Global Infotech; conflicting sameAs (x.com/imgglobal + linkedin/imgglobalinfotech vs twitter.com/imgglobalinfo + linkedin/img-global-infotech).
  • BreadcrumbList missing on 10/11 (DB/composer breadcrumb data empty; include is @if-gated). Only ai-in-retail renders it. No visible breadcrumb UI.
  • Service.areaServed="Global" on all 11 (some Org country arrays lead with US, India buried). Set India-first.
  • Service.provider is a loose name string, not an @id reference.
  • FAQPage NOT emitted on AI Development Company despite a visible 6-Q&A FAQ.
  • Malformed @id/URL with stray space (AI Chatbot, eCommerce, Healthcare).
  • Non-ISO dates (2025-1-13, US −08:00 offset) on Banking, Education, Healthcare, Retail — use zero-padded + IST +05:30.
  • Zero-width / non-joiner Unicode inside serviceType — AI Agent, LLM (48× U+200B), eCommerce (177 ZW codepoints). Strip + add build-time JSON-LD validation.
  • Service.name = full page title on Machine Learning, eCommerce — set a clean service name.

Content & E-E-A-T

  • Off-topic astrology / app-clone case-study block reused on all 6 service pages — critical on AI Agent (15 "astrology" + 40 "Mobile App Development" mentions). Replace with unique, on-topic AI/ML projects.
  • Fabricated / unsourced statistics on AI Agent ("3x DAU"), LLM, Education ("85%", "$10–25B"), Healthcare ("$187.95B / 37% CAGR" — YMYL), Retail ("USD 33.3T+"), eCommerce ("71%"). Cite a named source+year or remove.
  • Keyword stuffing / spun copy: Retail ("ai in retail" ×33), eCommerce trailing keyword clauses, ML "company india" fragments, Education "increase engagement in how is ai used in education".
  • USD-only pricing on India pages (AI Development "$8,000–$15,000", baked "From $8,000" inside schema descriptions; AI Chatbot USD under a "cost…in India" question). Show INR.
  • Duplicated trailing-word typos on AI Agent ("…workflows. or data workflows.") — in both the accordion and FAQPage JSON-LD.
  • Spelling: AI in Banking — "AI in Baking Development Process" (3 live occurrences, incl. image alt/title).
  • Weak author E-E-A-T: only the two founders appear as Person; no content author/reviewer. Add a real expert byline (priority on YMYL).

Images

  • Missing width/height (CLS): 24 imgs on most pages; 47 on AI Development; Machine Learning ships a CTA image with width="" / broken aspect-[].
  • Footer flag alt bug (global template): Canada flag/HQ images carry alt="Germany" — affects every page via the shared footer.
  • Positives: mostly WebP + lazy-loaded.

Internal linking

  • Zero <a href> links between any of the 11 pages. Sibling-slug mentions on AI Development are plain text, not anchors. Industry pages fully orphaned. No breadcrumb trail anywhere.

Technical & GEO

  • Heavy HTML: AI Agent 795 KB / 189 imgs, AI Development 759 KB, Generative AI 753 KB. Defer scripts, extend content-visibility:auto, trim repeated inline utility classes.
  • LCP not optimized: AI Agent hero is loading="lazy" fetchpriority="low" (only 1/189 eager). Set hero eager + high priority + preload.
  • Geo signals: en_US locale, zero hreflang, og:locale:alternate lists CA/GB/DE/AE but never IN. Thin India relevance in body (Banking: one RBI mention, no UPI/Indian banks).
