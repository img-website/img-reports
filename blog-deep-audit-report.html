🎯 #1 (Sabse bada) — Cache poisoning / wrong-page serving

Critical   Ek URL pe doosre blog ka content/schema serve ho raha hai

Multiple agents ne independently pakda — ek hi URL ko baar-baar fetch karne pe alag-alag pages serve hue:

  • custom-conversational-ai-bot-development-cost → kabhi hire-generative-ai, kabhi choosing-the-right-ecommerce-platform
  • doctor-on-demand-... ka BlogPosting schema → "Social Media App" ka
  • telemedicine-... ka saara schema → Social Media blog ka
  • society-management-... → "Mobile-First Ecommerce" ka schema
  • ai-agents-transforming-insurance → "Construction Management" ka schema
  • top-10-companies-to-hire-ecommerce-developers → "10 Best Ecommerce..." serve hua
  • from-idea-to-launch → "10 Best Ecommerce" ka schema

Impact: Google ko ek URL pe kabhi sahi, kabhi galat content/canonical/schema milta hai → confuse → wrong page index, conflicting canonicals, rank nahi, ya deindex. Yahi 7 "canonical points-elsewhere" aur kai wrong cannibalization-detections ki bhi asli wajah hai.

Root cause: Spatie ResponseCache galat cache-key pe pages serve kar raha (URL ke alawa kisi cheez pe keyed / race). Wahi ResponseCache jo /clearLaravelCache clear nahi kar raha tha.

Fix (P0): ResponseCache cache-key ko full URL pe set karo, poora cache clear, aur 5–10 URLs pe repeat-fetch test karo ki har URL apna hi content de.
